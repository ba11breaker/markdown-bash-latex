# Project 2: Write a Technical Blog Post

## Objective

Write a complete technical blog post in Markdown. Topic is your choice, but
it should be technical (e.g., a tutorial, how-to, or explainer).

## Requirements

Create a file called `my-blog-post.md` in this folder with:

1. **Title** as H1
2. **Metadata block** (date, author, tags — use a blockquote or table)
3. **Introduction** paragraph
4. **At least 3 content sections** with H2 headings
5. **Code examples** with syntax highlighting
6. **At least 1 image** (use a placeholder URL or local file)
7. **Inline code** references
8. **A summary/conclusion** section
9. **References/links** section

## Suggested Topics

- "Getting Started with Git"
- "Understanding CSS Flexbox"
- "How DNS Works"
- "Building a REST API with Python"
- "Introduction to Docker"

## Evaluation Criteria

- [ ] Clear, logical structure
- [ ] Proper use of headings (not too deep)
- [ ] Code blocks with correct language tags
- [ ] Engaging introduction
- [ ] Smooth transitions between sections
