# Project 1: Create a Professional README

## Objective

Create a README.md for an imaginary open-source project. This practices
all Markdown skills together.

## Requirements

Create a file called `my-project-readme.md` in this folder with:

1. **Project title** with a badge/shield image
2. **Description** (2-3 sentences)
3. **Table of Contents** with anchor links
4. **Installation** section with code blocks for multiple OS
5. **Usage** section with code examples
6. **API Reference** using a table
7. **Configuration** using a code block (YAML or JSON)
8. **Contributing** section with a task list
9. **License** section
10. **Acknowledgments** with links

## Bonus

- Add a Mermaid architecture diagram
- Add GFM alerts for important notes
- Add collapsible FAQ section
- Add badges at the top (use https://shields.io style)

## Evaluation Criteria

- [ ] All 10 required sections present
- [ ] Proper heading hierarchy (no skipped levels)
- [ ] At least 2 code blocks with syntax highlighting
- [ ] At least 1 table
- [ ] At least 1 list (ordered or unordered)
- [ ] At least 2 links
- [ ] Clean, consistent formatting
